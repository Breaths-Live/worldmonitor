# 🚀 Railway Relay Setup Guide

## What is Railway Relay?

The Railway relay server is a **multi-protocol gateway** that handles data sources requiring:
- Persistent WebSocket connections (AIS maritime)
- OAuth2 authentication (OpenSky aircraft)
- Residential proxying (OREF rocket alerts, Akamai WAF bypass)
- Telegram MTProto (OSINT channels)
- Request queuing (Polymarket API)

When **WS_RELAY_URL** is blank, 9 API domains are blocked (see [API_LIVE_CONNECTIONS.md](../API_LIVE_CONNECTIONS.md)).

## Option 1: Deploy on Railway.app (Recommended) ⭐

### **Step 1: Create Railway Account**

```bash
# Visit: https://railway.app
# Sign up with GitHub
# Create new project → Empty Project
```

### **Step 2: Deploy Relay Server**

```bash
# Clone the relay code (from worldmonitor repo)
git clone https://github.com/tncsharetool/worldmonitor.git relay-backup
cd relay-backup

# Connect to Railway
railway link

# Set environment variables in Railway dashboard:
# Settings → Environment Variables
# - WS_RELAY_URL=https://worldmonitor-relay.up.railway.app
# - RELAY_SHARED_SECRET=$wm_relay_2024_secure_shared_secret_coachchuyen
# - AISSTREAM_API_KEY=d901004ff3eced23bc5d0d942f0a009235ba8928
# - OPENSKY_CLIENT_ID=coachchuyen-api-client
# - OPENSKY_CLIENT_SECRET=AnB3YU1LnxEeTe39baFYlLb4SFfOjhjK
# - TELEGRAM_API_ID=21575411
# - TELEGRAM_API_HASH=a3dbd4a4d8a6a24ff1c1eee5e5c8d9e7

# Deploy
railway up

# View logs
railway logs
```

### **Step 3: Update Local .env.local**

```bash
# In .env.local:
WS_RELAY_URL=https://worldmonitor-relay.up.railway.app
VITE_WS_RELAY_URL=wss://worldmonitor-relay.up.railway.app
RELAY_SHARED_SECRET=$wm_relay_2024_secure_shared_secret_coachchuyen
```

## Option 2: Run Relay Locally (Development) 

### **Step 1: Install Dependencies**

```bash
npm install
```

### **Step 2: Generate Relay Secrets**

```bash
# Generate a random 32-char hex token
RELAY_SECRET=$(openssl rand -hex 24 | sed 's/^/wm_/')
echo "RELAY_SHARED_SECRET=$RELAY_SECRET" >> .env.local
```

### **Step 3: Start Relay Server**

```bash
# Terminal 1: Start relay server
node scripts/ais-relay.cjs

# Should output:
# ✅ AIS Relay WebSocket: ws://localhost:8080
# ✅ HTTP API: http://localhost:8081
# ✅ gzip compression: enabled
```

### **Step 4: Update .env.local for Local Development**

```bash
# In .env.local:
WS_RELAY_URL=http://localhost:8081
VITE_WS_RELAY_URL=ws://localhost:8080
```

### **Step 5: Start Dev Server (Terminal 2)**

```bash
npm run dev

# Should see:
# ✅ VITE v6.4.1 ready in 966 ms
# ✅ Local: http://localhost:3000/
```


## Environment Variables for Relay

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `AISSTREAM_API_KEY` | Yes | AIS vessel tracking | `d901004ff3eced...` |
| `OPENSKY_CLIENT_ID` | Yes | Military aircraft OAuth2 | `coachchuyen-api-client` |
| `OPENSKY_CLIENT_SECRET` | Yes | OpenSky secret | `AnB3YU1LnxEe...` |
| `RELAY_SHARED_SECRET` | Yes | HMAC auth token | `$wm_relay_2024...` |
| `TELEGRAM_API_ID` | No | Telegram app ID | `21575411` |
| `TELEGRAM_API_HASH` | No | Telegram app hash | `a3dbd4a4d8a6...` |
| `TELEGRAM_SESSION` | No | GramJS auth session | (generated by script) |
| `CLOUDFLARE_API_TOKEN` | No | Internet outages | `b_ywynnJ-hG5y...` |

## Troubleshooting

### **Relay Not Starting**

```bash
# Check Node.js version
node --version  # Should be 18+

# Check port 8080/8081 availability
lsof -i :8080
lsof -i :8081

# Kill blocking process if needed
kill -9 <PID>
```

### **WebSocket Connection Fails**

```bash
# Verify relay is listening
netstat -an | grep 8080

# Check firewall (if running on remote server)
sudo ufw allow 8080/tcp
```

### **AIS Data Not Updating**

```bash
# Verify AISSTREAM_API_KEY is correct
curl -i 'https://stream.aisstream.io/v0/stream?subscribe-key=$AISSTREAM_API_KEY&mmsi=0'

# Should return 400 (invalid MMSI) not 401 (invalid key)
```

### **OpenSky Authentication Fails**

```bash
# Test OAuth2 token via basic auth
curl -X POST 'https://opensky-network.org/api/v1/tokens/validate' \
  --basic -u "coachchuyen-api-client:AnB3YU1LnxEe..."

# Should return: {"username": "coachchuyen-api-client", ...}
```
